## 2024-03-07 - React.memo and Stable References
**Learning:** In React, when using `React.memo` to optimize child components (like `HacksSection` and `HackCard`), the memoization is broken if the parent component passes down inline arrow functions or un-memoized functions as props. This is because a new function reference is created on every render, causing the child to re-render despite `React.memo`. This is specifically important in this codebase when dealing with mapping over lists and passing down handlers that require an `id`.
**Action:** When memoizing a list component with `React.memo`, ensure all functions passed as props are wrapped in `useCallback`. Furthermore, for list items, the child component should accept the generic stable callback and the item's `id`, and call the stable callback with the `id` internally, rather than creating an inline arrow function in the list map.
