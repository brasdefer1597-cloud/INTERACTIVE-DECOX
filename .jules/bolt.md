## 2024-05-18 - Missing Memoization in Mapped List Items causes excessive Re-renders
**Learning:** In a highly interactive section (`HacksSection`) rendering a grid of items, mapping through data and passing inline functions to child components (`HackCard`) breaks memoization. React will re-render all children every time the parent re-renders due to referential inequality of the inline functions, even if the props of those children didn't semantically change. This is especially impactful in list rendering.
**Action:** When a parent component renders a list or grid of child components that accept event handlers, always use `React.useCallback` in the parent to create stable references for the handlers. Pass an identifier (like `id`) to the handler so you don't need to create a new closure for each item. Then wrap the child component in `React.memo` to skip rendering when props remain equal.
