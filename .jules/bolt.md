## 2024-03-17 - React.memo on Lists requires stable callbacks
**Learning:** When trying to prevent unnecessary re-renders in a large list of components, wrapping the list item (e.g. `HackCard`) in `React.memo` is completely ineffective if the parent component passes inline functions as props (e.g. `onActivate={() => onActivateClick(id)}`). The inline functions create new references on every render of the parent, causing `React.memo` to consider the props as changed and triggering a full re-render of every item in the list, completely negating the memoization.
**Action:** In order to use `React.memo` effectively in list items, change the item component to accept the specific ID (or object) in the callback (e.g. `onActivate: (id: number) => void`), and use `React.useCallback` in the parent to create a single, stable callback function to pass down to all children. The children can then pass their specific ID back up to the parent via the stable callback.
